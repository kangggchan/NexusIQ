-Goal-
You are analyzing operational data for NovaDrive AI's microservices platform (NexusIQ).
Given a text document — which may be an incident report, deployment log, Slack thread,
Jira ticket, post-mortem, runbook, or architecture doc — identify all entities of the
specified types and all relationships among them.

-Steps-
1. Identify all entities. For each identified entity, extract the following information:
- entity_name: Name of the entity, capitalized (e.g., PAYMENT-SERVICE, INC-2024-001, ALICE CHEN)
- entity_type: One of the following types: [{entity_types}]
- entity_description: Comprehensive description of the entity's attributes, role, and
  operational status. For services, include owner team, dependencies, and SLO data if
  available. For incidents, include severity, timeline, and affected systems.
Format each entity as ("entity"{tuple_delimiter}<entity_name>{tuple_delimiter}<entity_type>{tuple_delimiter}<entity_description>)

2. From the entities identified in step 1, identify all pairs of (source_entity, target_entity)
that are *clearly related* to each other. Focus on operational relationships:
- service-to-service dependencies (synchronous calls, async events, shared databases)
- incident-to-service impact relationships
- deployment-to-incident causation
- team ownership of services or incidents
- commit / PR changes that triggered or contributed to failures
For each pair of related entities, extract the following information:
- source_entity: name of the source entity, as identified in step 1
- target_entity: name of the target entity, as identified in step 1
- relationship_description: explanation of the operational relationship (e.g.,
  "PAYMENT-SERVICE calls AUTH-SERVICE synchronously on every checkout request")
- relationship_strength: a numeric score 1-10 indicating operational coupling strength
Format each relationship as ("relationship"{tuple_delimiter}<source_entity>{tuple_delimiter}<target_entity>{tuple_delimiter}<relationship_description>{tuple_delimiter}<relationship_strength>)

3. Return output in English as a single list of all the entities and relationships
identified in steps 1 and 2. Use **{record_delimiter}** as the list delimiter.

4. When finished, output {completion_delimiter}
